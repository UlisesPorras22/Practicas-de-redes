------------------------------------NUMERADAS----------------------------------
" Comando ALC estándar numerada IPv4 "
access-list (1 a 99 o 1300 a 1999) {deny | permit | remark text} source [wildcard] [log]

deny-----------Esto deniega el acceso si la condición coincide.
permit---------Esto permite el acceso si la condición coincide.
remark text----(Comentarios) Se agrega una entrada de texto para fines de documentación es opcional.

Soucrce--------Esto identifica la red de origen o la dirección de host que se va a filtrar
		any para especificar todas las redes
		host para identificar una dirección IP específica(o sin escribir host, solo la ip)

Wildcard para aplicar al origen. Si se omite, se asume una máscara 0.0.0.0 predeterminada.
		ejemplo---permit 192.168.20.0 0.0.0.255  || permit host 192.168.10.10

access-list (100 a 199 o 2000 a 2699) {deny | permit | remark text} (protocol) (source) [wildcard] [operator {port}] destination destination-wildcard [operator {port}] [established] [log]

protocol--------Nombre o número de un protocolo de Internet
destination-----Esto identifica la red de destino o la dirección de host que se va a filtrar.Any o host
port------------Nombre o numero del puerto(opcional)
established-----Sólo para el protocolo TCP

" ACL extendido  si el puerto del host de destino es 80"http"
access-list 100 permit tcp 192.168.10.0 0.0.0.255 any eq [www || 80]
